<h1>Meu Plantão - Sistema gerenciador de plantões médicos</h1>

O Hospital Beneficente Unimar (HBU) realizou uma apresentação em sala de aula na
qual conseguimos elaborar o levantamento de requisitos. Durante a apresentação,
foram mostradas várias propostas de sistemas, dentre as opções apresentadas, nós
optamos pelo desenvolvimento de um Sistema de Gestão de Plantões, utilizando
como base o funcionamento geral de plataformas existentes voltadas para
intermediação de plantões médicos.

## Problemas percebidos
Também foi informado que o hospital já utiliza um sistema de gestão de plantões,
porém esse sistema não é próprio, e por tanto, alguns problemas ainda podem ocorrer:
<ul>
  <li>Dependência de um sistema de terceiros.</li>
  <li>Dificuldade de personalização conforme as necessidades do hospital.</li>
  <li>Possíveis limitações de funcionalidades.</li>
  <li>Falta de integração com outros processos internos.</li>
  <li>Processos que ainda podem ocorrer manualmente, como comunicação de
  trocas de plantão.</li>
</ul>

Também é possível que essas limitações podem gerar retrabalho administrativo ou
dificultar melhorias operacionais.

## Objetivos do projeto
Com este projeto, pretendemos desenvolver um sistema que:

<ul>
  <li>Simule uma solução própria de gestão de plantões para o hospital.</li>
  <li>Centralize o gerenciamento das escalas.</li>
  <li>Permita o controle de trocas de plantão.</li>
  <li>Atenda de forma mais específica às necessidades instanciadas.</li>
  <li>Processos que ainda podem ocorrer manualmente, como comunicação de
  trocas de plantão.</li>
</ul>

A proposta é criar um sistema web funcional dentro do contexto acadêmico,
representando uma alternativa personalizada ao sistema atualmente utilizado

## Tecnologias Utilizadas

<ul>
  <li>Front-end => <strong>React</strong></li>
  <li>Back-end => <strong>ASP.NET</strong></li>
  <li>Banco de Dados => <strong>PostgreSql</strong></li>
  <li>Documentação => <strong>Swagger API</strong></li>
</ul>
